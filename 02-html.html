Section 02 — Markup

HTML

HyperText Markup Language is the backbone of the web. It defines the structure and meaning of content. Below are live demonstrations of key HTML concepts.

Semantic Document Structure HTML5 SEMANTICS
What this demonstrates: HTML5 introduced semantic elements that give meaning to page structure. Unlike using generic <div> tags, semantic elements like <header>, <nav>, <main>, <article>, <aside> and <footer> describe the purpose of each section — improving accessibility, SEO, and code readability.

HTML Forms & Input Types FORMS
What this demonstrates: HTML forms collect user input. The <form> element wraps inputs, which each have a type attribute (text, email, password, date, select etc.). Labels are associated with inputs via the for and id attributes, which is essential for accessibility. The required, placeholder and pattern attributes provide built-in validation.
HTML Tables TABULAR DATA
What this demonstrates: Tables are used to display tabular data in rows and columns. A properly structured table uses <thead> for column headings, <tbody> for data rows, and optionally <tfoot> for summaries. The scope and caption attributes improve accessibility for screen readers.

HTML Media Elements MEDIA
What this demonstrates: HTML5 introduced native media elements — <img>, <video>, <audio> and <figure> — allowing media to be embedded directly without plugins. The alt attribute on images is critical for accessibility. The controls attribute on video/audio adds native playback UI.

Accessibility Attributes A11Y
What this demonstrates: HTML provides built-in attributes to make web pages accessible to people using assistive technologies like screen readers. ARIA (Accessible Rich Internet Applications) attributes extend native HTML accessibility when needed.
  • 🏷️

    alt attribute

    Provides text description for images. Used by screen readers when images cannot be seen. <img alt="Description">

  • 🔗

    label + for

    Associates a label with its form input so screen readers announce the label when the field is focused. <label for="email">

  • 📐

    role & aria-label

    ARIA attributes describe the role or label of elements when semantics alone aren't enough. role="button" aria-label="Close"

  • ⌨️

    tabindex

    Controls keyboard navigation order. Ensures users who navigate via keyboard can reach all interactive elements. tabindex="0"
